# MVP 设计参考

在产品发现流程中，MVP（最小可行产品）的设计至关重要。本文件提供如何划定功能边界、确定核心闭环及成功指标的指导，帮助团队在有限资源下快速验证产品价值。

## 确定功能边界

在设计 MVP 时，先确定哪些 **不做**，再决定 **做什么**。

1. **不做什么**：列出当前阶段不应涉及的功能或模块，例如：
   - 复杂的用户权限系统
   - 高度定制化的工作流或可视化配置器
   - 支撑多语言、跨区域的国际化需求
   - 大规模数据迁移或复杂集成
2. **必须做**：支撑产品核心闭环的关键功能，缺一不可。例如：
   - 用户输入/数据采集
   - 关键算法或核心服务
   - 结果展示与反馈
3. **可以做**：提升用户体验、验证某些假设但非必需的功能，例如：
   - 个性化推荐
   - 分享功能
   - 高级筛选与搜索
   - 数据导出

通过将功能划分为“必须做”“可以做”“暂时不做”，可帮助团队保持重点，避免功能膨胀。

## 设计核心闭环

MVP 的核心闭环是最小可行产品中必须跑通的流程，通常包括以下环节：

1. **触发/输入**：用户发起操作或系统自动捕获数据的方式。
2. **处理/核心能力**：产品完成核心价值交付的过程，如智能推荐、数据汇聚、算法计算等。
3. **输出/反馈**：向用户展示处理结果或动作建议。反馈应直观并促使用户采取下一步行动。
4. **学习/优化**（可选）：系统根据反馈或数据进行迭代优化，例如模型训练、内容更新等。

例如，对于一个帮助研究员生成行业报告的 AI 工具，核心闭环可能是：

> 研究员输入公司或行业关键词 → 系统抓取公开数据并生成初步洞见 → 用户审核与编辑 → 系统根据用户反馈调整分析重点。

在设计 MVP 时，请明确闭环中每一步需要哪些功能支撑。

## 成功指标

定义好成功指标有助于判断 MVP 是否达到预期效果。常见指标包括：

1. **激活指标**：首次使用或注册后的关键行为，如完成首个任务或触发核心功能。
2. **使用频率**：用户在一定时间内使用产品的次数或时长，体现粘性。
3. **任务完成率**：核心任务的完成度，如报告生成成功率或工作流闭环率。
4. **留存**：第 X 天或第 X 周留存率，衡量用户持续使用意愿。
5. **转化**：付费或付费意向（如果有付费模式），或向其他业务流程的转化率。
6. **用户满意度**：可通过问卷评分、NPS、评论、用户访谈等方式收集。

MVP 阶段不需要用所有指标，建议选择 2–3 个最能反映价值假设的指标进行追踪和评估。

## 示例输出结构

在报告或讨论中描述 MVP，可以参考以下结构：

```
### 产品不做什么
- …
- …

### MVP 核心闭环
1. 用户触发：……
2. 系统处理：……
3. 结果输出：……
4. （可选）持续优化：……

### MVP 功能分类
- **必须有**：……
- **可以有**：……
- **暂时不做**：……

### 成功指标
- 激活指标：……
- 使用频次指标：……
- 任务完成指标：……
- 留存/转化/满意度指标：……
```

通过清晰地划分功能、闭环和指标，团队可以集中资源，在最短时间内验证核心假设。
