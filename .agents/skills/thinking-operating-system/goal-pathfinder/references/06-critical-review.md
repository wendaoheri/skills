# 批判审查参考

在目标规划过程中，需要从批判者视角识别隐藏风险，防止自欺和自嗨。此文件提供批判审查的目标、常见自欺模式、主要方法以及结构化输出模板。

## 批判审查目标

1. **揭示过度乐观**：识别用户对资源、执行能力和外部环境的高估。
2. **指出隐藏假设**：发现被忽略或不合理的前提条件，如用户行为、技术可行性或市场反应。
3. **预判失败原因**：通过假设目标失败进行逆向分析，找出最可能导致失败的因素。
4. **设定止损与调整**：在方案执行前设定预警机制，防止投入过多资源而未能达成结果。

## 常见自欺与自嗨模式

- **高估资源**：以为拥有某项资源或能力，但实际上掌握有限，或需要长时间才能获得。
- **低估难度**：忽视技术、市场或执行的复杂性，把事情想得过于简单。
- **忽视基准概率**：只关注成功案例，不考虑多数人的失败经验。
- **替代执行**：把准备工作、研究、学习当作执行，而迟迟不采取实际行动。
- **宏大叙事**：用华丽的愿景掩盖缺乏细节的执行策略。
- **确认偏差**：只寻找支持自己观点的信息，忽视相反证据。
- **外部归因**：对失败归因于外部客观条件，忽视自身问题。
- **假装忙碌**：用细枝末节填满时间，避免面对真正重要但困难的任务。

## 关键方法

### Red Team / Devil’s Advocate

由另一个独立“团队”或角色模拟反方，从最苛刻的角度审查目标和计划。要像“敌人”一样挑剔，指出漏洞、风险和不合理假设。

### Pre‑mortem（预先假死）

假设某个目标或项目已经失败，设想导致失败的最可能原因。逆向思考使人更容易识别风险，而不是停留在乐观的期望中。

### Inversion（反向思考）

思考如果要让目标失败，应当做哪些事情。将这些失败行为列出，并在执行计划中避免发生。

### Base Rate Thinking（基准概率思维）

查找类似目标在群体中的成功和失败比例，预估成功概率和常见失败原因。以现实数据为基准，减少过度自信。

## 输出模板

在批判审查阶段，可以使用以下结构输出：

```
### 最强反方观点

用一段话（或多条要点）阐述从反方角度看，这个目标或路径为什么可能是错误的或无意义的。不要为了鼓励用户而软化措辞。

### 你最可能高估的地方
- …

### 你最可能低估的地方
- …

### 你可能在自欺的地方
- …

### 最危险的隐藏假设
- …

### 如果目标失败，最可能原因
- …

### 必须设置的止损条件
- 如果出现以下情况，则应停止或调整路径：……
- 止损阈值可以是时间、资源消耗、关键指标等。
```

通过直面可能的失败和风险，可以在方案执行前及时修正，提高成功概率，并避免无谓的投入和自我欺骗。
