# 执行计划参考

在完成目标澄清、背景挖掘、现实校验、路径设计和个人适配后，需要制定具体的执行计划。本文件提供制定执行计划的原则、阶段设计以及模板，帮助将选择的路径转化为实际行动。

## 执行计划原则

1. **阶段明确**：将长期目标拆分为短期、可管理的阶段，例如 2 周、1 个月、3 个月等。
2. **验证优先**：在计划中嵌入验证环节，避免在未验证假设的情况下投入过多资源。
3. **指标驱动**：为每个阶段设定输入、过程和结果指标，衡量进度和成效。
4. **复盘机制**：安排定期复盘，总结经验、调整策略和下一步计划。
5. **止损与调整**：为计划设定止损阈值，例如时间、预算、或关键指标，当达不到预期时及时调整或停止。
6. **最小行动**：优先列出最小可行行动（Minimum Viable Action），确保第一步简单明确，降低启动阻力。

## 阶段设计示例

以下是一个常见的三阶段设计，仅作参考，可根据目标和背景调整：

### 第一阶段：澄清与验证（周期：1–2 周）

**目标**：验证目标是否值得继续投入，收集真实反馈，明确可行路径。

**动作**：

- 完成对若干名相关方（客户、专家、导师）的访谈或调研。
- 制作简易原型或示意图，验证用户反应或市场需求。
- 检查关键假设是否成立，识别最大未知数。
- 梳理所有必要资源和预期障碍。

**指标**：
- 访谈数量、反馈质量、正向信号强度。
- 原型或示意图反馈的满足度或吸引力评分。

### 第二阶段：小规模试跑（周期：2–4 周）

**目标**：构建最小闭环，获取初步成果和执行经验，发现执行障碍。

**动作**：

- 构建 MVP 或执行试点，覆盖核心功能或关键路径。
- 收集试点数据，验证价值假设和用户反馈。
- 记录执行过程中的问题和改善机会。
- 调整闭环，准备放大执行。

**指标**：
- 核心任务完成率、用户参与度、满意度或留存率。
- 闭环运行次数、成功率。

### 第三阶段：放大投入（周期：1–3 个月）

**目标**：建立稳定节奏，扩大战果，逐步完善和扩展。

**动作**：

- 根据试点结果优化产品或方案，增加功能或扩大范围。
- 建立持续交付和反馈机制。
- 若有商业目标，开始收费试验或寻找付费客户。
- 开展市场或渠道推广。

**指标**：
- 用户增长、收入或成本收益比。
- 产品迭代次数、功能扩展范围。
- 指标达成率和用户满意度。

## 行动计划模板

在汇报或输出时，可以使用以下模板来描述执行计划：

```
### 未来 14 天
- 第 1–3 天：……
- 第 4–7 天：……
- 第 8–14 天：……

### 未来 30 天
- …

### 未来 90 天
- …

### 输入指标
- 资源投入、时间投入、行动次数等。

### 过程指标
- 工作进度、任务完成率、用户参与度等。

### 结果指标
- 用户满意度、转化率、收入或成本节约等。

### 复盘机制
- 每周/每两周举行复盘会议。
- 总结本阶段达成情况、发现的问题和下一步计划。

### 风险预案
- 如果出现以下情况，则采取调整或止损：……
- 如果连续 X 周无进展，重新评估目标或路径。
- 如果关键假设被证伪，则停止或转向其他路径。
```

通过采用分阶段、可验证的执行计划，用户可以更有效地推进目标，同时根据反馈不断优化路径，减少无效投入。
