# 现实校验参考

此文件用于帮助评估用户目标在外部环境中的可行性。通过分析行业、市场、政策、技术等外部因素，并使用基准概率和第一性原理等方法，识别机会、阻力和必须验证的事实。

## 现实校验目标

现实校验的目的是确保目标不是基于幻想或脱离现实的期望。它关注的不仅仅是用户自身条件，还包括目标所处的外部环境，例如：

- 市场规模与增长趋势。
- 行业发展与竞争格局。
- 政策法规与监管要求。
- 技术发展路径与可行性。
- 经济、社会、文化变化的影响。
- 典型成功与失败案例。

## 校验维度

1. **机会与趋势**：外部环境中有哪些积极因素利于达成目标？例如市场增长、政策支持、技术突破、社会意识变化等。
2. **阻力与风险**：外部环境中有哪些阻碍因素？例如竞争激烈、监管限制、用户认知不足、经济周期、供应链风险等。
3. **时间窗口**：现在是否是合适的时间？未来可能更好还是错过窗口？
4. **可复制性与壁垒**：目标的核心能力是否易于复制？是否存在先发优势或数据壁垒？
5. **合规与伦理**：目标的实现是否涉及法律或伦理风险，如数据隐私、金融监管、医疗安全等。
6. **必备资源与外部依赖**：达成目标需要哪些外部资源或合作伙伴？这些资源是否易于获得？

## 搜索与验证提示

当目标涉及以下方面时，应主动说明需要搜索或验证，并在工具可用时执行：

- **最新行业趋势或政策变化**：例如技术迭代速度、监管动态、市场用户需求变化等。
- **市场规模与竞争格局**：了解目标领域的市场容量、潜在用户、主要竞争对手及其优势。
- **技术可行性**：关键技术是否成熟？有哪些已知的技术瓶颈或解决方案？
- **成功与失败案例**：类似目标有无公开案例，可以借鉴或对比？
- **成本与价格**：实现目标需要的投入成本，对标产品或服务的定价。
- **法律法规与风险**：如涉及敏感行业（医疗、金融、教育等），必须查阅相关法规与合规要求。

## 输出结构

在汇报或阶段小结中，可以采用以下结构输出现实校验结果：

```
### 外部现实判断

- 机会：……
- 阻力：……
- 时间窗口：……
- 可复制性与壁垒：……
- 合规与伦理风险：……
- 必备资源与外部依赖：……

### 必须验证的信息

- …

### 初步现实结论

基于当前信息，判断目标在外部环境下：
- 是否值得继续推进。
- 是否需要降级或调整。
- 是否需要先进行验证或试点。
- 是否存在明显高风险，需要重新考虑。
```

通过对外部环境进行系统分析，可以帮助用户避免过度乐观或孤立思考，从而制定更加实际的目标路径。
