# 思维操作系统

## 概述

**思维操作系统**是一套系统性应对复杂问题的技能套件，核心理念是：**定义问题往往比解决问题更重要**。它将第一性原理分析、叙事框架、问题拆解、用户洞察、结构化决策、清晰表达和批判性审查融为一体，为高风险项目提供可复用的工作流。同时，它还包含一套学习设计技能，支持学习者分析、目标设计、叙事构建、案例设计和幻灯片制作，帮助你不仅解决复杂问题，还能有效传达思路并教授他人。

## 适用场景

在以下情况调用此元技能：

- 启动目标不明确或利益相关方意见分歧的新项目。
- 需要将零散想法或研究成果转化为结构化方案。
- 决策涉及多重标准、权衡取舍和长期影响。
- 需要向高管或客户传达复杂的推理过程。
- 需要就某个复杂问题进行多角度深入讨论，并形成结构化结论。

## 工作流程

0. **目标路径探索** (`goal-pathfinder`) *（可选前置步骤）*  
   当目标本身尚不清晰或未经验证时，在一切分析之前先使用此技能。澄清目标背后的真实动机，评估个人限制与外部现实，设计多条达成路径，并从批判者视角压力测试方案。产出结构化的目标路径分析报告。

1. **第一性原理分析** (`first-principles`)  
   从基本事实出发挑战假设，重新定义问题。判断当前问题框架是否成立，必要时进行重新定框。

2. **SCQA 问题定义** (`scqa-problem-definition`)  
   描述现状（Situation）、点明复杂性（Complication）、提出核心问题（Question）、预告简要答案（Answer），让利益相关方对"我们在解决什么"达成共识。

3. **议题树构建** (`issue-tree-builder`)  
   将 SCQA 的核心问题拆解为 MECE 子问题层级，绘制分析地图，确保全面覆盖。

4. **JTBD 分析** (`jtbd-analysis`)  
   深挖议题树各分支所对应的用户任务（Jobs to be Done），明确功能性与情感性任务，为方案设计提供依据。

5. **产品发现** (`product-discovery`) *（可选，产品赛道）*  
   当任务涉及产品构建或验证时，在 JTBD 分析之后使用。运行结构化的产品发现流程：澄清产品想法、定义核心问题、梳理用户分层与场景、确定定位、分析竞品、划定 MVP 边界、挖掘盲点，并产出验证计划与产品孵化报告。

6. **决策矩阵** (`decision-matrix`)  
   当多个方案浮现时，按加权标准（如影响力、投入、风险）进行比较，选出最优路径并记录权衡取舍。

7. **金字塔结构** (`pyramid-structure`)  
   将分析结论组织为清晰论点：以核心结论开头，依次呈现有逻辑顺序的论据与支撑材料。

8. **批判性审查** (`critic-reviewer`)  
   在定稿前，审视逻辑、证据和假设，识别漏洞，打磨输出，使其更加严密有说服力。

9. **深度讨论** (`deep-discussion`) *（可选，复杂问题研讨）*  
   当问题需要多角度反复论证、或需要在收敛前充分暴露分歧与盲点时使用。通过主持人、导师、对手三种角色的结构化互动，识别核心张力（认知分歧 / 事实待查 / 框架差异），并在讨论结束时生成结构化报告。

如果结论需要对外传达或转化为培训内容，继续以下步骤：

10. **受众分析** (`audience-analysis`)  
    收集学习者的人口统计、角色和态度信息，对受众分层，调整内容基调。

11. **学习目标设计** (`learning-objective-designer`)  
    运用布鲁姆分类、ABCD 和 SMART 框架，将宏观目标转化为具体可衡量的学习成果。

12. **培训叙事构建** (`training-storyline-builder`)  
    将内容打造成引人入胜的叙事旅程（如英雄之旅），并规划异步、直播与文档化等不同交付形式。

13. **案例演示构建** (`case-demo-builder`)  
    设计能够呈现权衡取舍、促进批判性思维的案例或模拟练习，提供情景、任务、讨论问题和复盘引导。

14. **幻灯片素材构建** (`slide-material-builder`)  
    将大纲转化为完整幻灯片。输出幻灯片提纲，并使用 WebSlides 和设计最佳实践生成 HTML 文件。

## 输出格式

作为元技能，本技能不产出单一输出，而是按顺序协调各子技能的调用，并汇总其结果。典型工作流产出：

0. 目标路径分析报告——来自 **goal-pathfinder**（当目标需要验证时）。
1. 重新定框后的问题陈述——来自 **first-principles**。
2. SCQA 叙事——来自 **scqa-problem-definition**。
3. 议题树大纲——来自 **issue-tree-builder**。
4. JTBD 洞察与任务陈述——来自 **jtbd-analysis**。
5. 产品孵化报告——来自 **product-discovery**（产品赛道时）。
6. 决策矩阵表格与推荐方案——来自 **decision-matrix**。
7. 金字塔结构论点摘要——来自 **pyramid-structure**。
8. 批判性审查报告——来自 **critic-reviewer**。
9. 结构化讨论报告——来自 **deep-discussion**（复杂问题研讨时）。

包含培训步骤时，还可能产出：

10. 每个学习者群体的受众画像摘要。
11. 含受众、行为、条件、程度的学习目标列表。
12. 含阶段、目的、活动和交付方式的叙事大纲。
13. 含情景、任务、引导方式、讨论问题和复盘说明的案例包。
14. 幻灯片大纲及生成的 HTML 幻灯片文件路径。

## 说明

- 每个子技能均可独立调用；但组合使用能产生更深的洞察和更高质量的输出。
- 灵活裁剪流程：简单任务可跳过决策矩阵或议题树；用户研究任务可迭代 JTBD 分析；需要充分辩证的复杂问题可插入深度讨论。
- "思维操作系统"的价值在于强制执行有纪律的思考，降低在未理解问题的情况下就追求解决方案的风险。
