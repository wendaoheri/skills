# 用户与场景拆解参考

在产品发现流程中，了解目标用户和使用场景是至关重要的一步。本文件提供分层用户分析、场景地图模板以及如何选择最高价值场景的指导。

## 用户分层

将用户区分为不同角色有助于识别产品的价值链。典型的角色包括：

| 角色         | 描述                                                         |
|--------------|--------------------------------------------------------------|
| **核心用户** | 最直接使用产品的人，他们的需求最能体现产品价值。               |
| **次要用户** | 会间接使用产品或受产品影响的人，功能需求较弱或较次。           |
| **影响者**   | 虽不直接使用产品，但会影响其他用户的决策，例如同事、行业专家。 |
| **付费者**   | 支付产品费用的人或部门，在企业场景中尤为重要。               |
| **决策者**   | 决定是否采纳产品的人，可能是管理层或部门领导。               |

根据具体产品类型，还可以添加：运维人员、支持人员、监管人员等角色。明确每个角色的需求和痛点，有助于避免遗漏关键利益相关方。

## 场景地图

在理解用户之后，需要将他们放在具体场景中观察。场景地图可以帮助团队系统性地分析用户在何种情境下会使用产品、他们的目标、当前做法和痛点是什么，从而发现产品机会。

建议采用表格形式，例如：

| 场景               | 用户目标               | 当前做法                 | 痛点                         | 产品机会                   |
|--------------------|------------------------|--------------------------|------------------------------|----------------------------|
| 信息收集与整理     | 快速获取最新行业信息   | 订阅邮件、手动整理 Excel | 信息分散、耗时长            | 自动聚合信息并智能整理     |
| 行业趋势分析       | 理解竞争对手动态       | 阅读报告、人工对比数据   | 数据来源零散、缺乏实时性    | 自动数据抓取与趋势分析     |
| 报告生成           | 输出高质量研究报告     | 重复搜索资料、手工撰写   | 写作效率低、容易遗漏重要点  | 智能辅助撰写并查缺补漏     |

每个产品团队应根据自身业务场景定义列项。通常建议包括：**场景名称**、**用户目标**、**当前做法**、**痛点**、**产品机会**。必要时可增加“使用频率”“现有工具”“影响范围”等列。

## 选择最高价值场景

评估哪些场景最有价值时，可以考虑以下因素：

1. **痛点强度**：场景中的痛点越强烈，用户越愿意尝试新方案。
2. **频次和规模**：发生频率高、涉及用户多的场景带来的价值潜力更大。
3. **差异化空间**：现有方案的可替代性和改善空间。若现有方案表现很差，那么改进机会更大。
4. **数据/能力闭环**：场景是否能形成数据或能力的自循环，为后续功能扩展提供基础。
5. **商业价值**：场景是否与核心商业目标一致，是否方便定价或收费。

综合以上因素挑选 1–2 个最有价值的场景，用于后续的 MVP 设计。这些场景应该代表核心用户最迫切的问题，同时在技术和资源允许范围内相对容易实现。
