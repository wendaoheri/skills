# 路径设计参考

在确定目标和背景后，需要设计多条可行路径而非单一路径。本文件提供使用 Theory of Change、Backcasting、决策树和情景规划等方法设计路径的指南，并给出路径类型和比较维度。

## 路径设计原则

1. **多路径选择**：至少设计稳健路径、进取路径和低成本试错路径，避免单线作战。
2. **因果链思维**：明确达成目标所需的中间结果和前置条件，找出每一步的关键要素。
3. **倒推规划**：从最终状态反推所需的中期里程碑和近期行动，确保路径可落地。
4. **成本收益分析**：每条路径需要衡量投入成本、预期收益、执行难度、风险水平和时间周期。
5. **情景考量**：考虑不同外部情景（乐观、中性、悲观）下路径的表现，评估适应性。

## 方法概述

### Theory of Change

该方法帮助拆解目标背后的因果链。具体步骤：

1. 确定最终目标。
2. 列出必须实现的中间目标（中期成果）。
3. 确定每个中间目标的前置条件和必要活动。
4. 分析这些条件现在是否具备，是否需要额外资源或验证。
5. 绘制因果链或逻辑模型，作为路径设计基础。

### Backcasting

该方法强调从未来的理想状态倒推。步骤：

1. 描述目标达成时的清晰状态。
2. 倒推到半程（例如一半时间），设想此时应该完成哪些里程碑。
3. 再倒推到最近的时间点（例如一个月），思考当前必须完成哪些任务。
4. 以倒推方式制定行动计划，确保每一步都指向最终目标。

### 决策树与情景规划

在有多个选择或不确定性较高的情况下，可以构建决策树：

1. 列出主要决策节点，如投入多或少、选择不同市场、采用不同技术等。
2. 评估每个节点的可能结果，标注概率、收益和风险。
3. 结合情景规划设计乐观、中性、悲观等情境，分析路径在不同环境下的表现。

通过这类分析，可以量化或至少结构化地比较不同路径。

## 路径类型模板

通常可设计以下三种路径：

1. **稳健路径**：
   - 适合风险承受能力一般、时间有限、需要兼顾其他责任的用户。
   - 优先确保稳步推进和安全边际，可能牺牲速度或规模。
2. **进取路径**：
   - 适合动机强烈、资源充足、可承受较高风险的用户。
   - 追求快速达成目标或更大的成果，但执行压力大，失败代价高。
3. **低成本试错路径**：
   - 适合目标尚未充分验证、用户资源有限或不确定性较高的情况。
   - 通过小步试错快速验证方向，不追求一次成功，成本较低。
