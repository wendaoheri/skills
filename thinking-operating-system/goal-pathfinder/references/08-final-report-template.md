# 目标路径分析报告模板

当用户完成了目标澄清、背景挖掘、现实校验、路径设计、个人适配、批判审查和执行计划制定后，可以根据本模板生成完整报告。报告应客观、结构化地汇总分析和决策，为用户后续执行或汇报提供依据。

## 报告使用注意

- 报告中的内容应区分已确认事实、合理推断和待验证假设。
- 对不确定的地方标注“待验证”，避免给人确定性的错觉。
- 如果报告用于向他人汇报，应适当精简细节，突出结论和建议。

## 报告结构

### 1. 一句话结论

概括目标是否值得推进以及推荐路径。

### 2. 目标概述

- **用户目标**：澄清后的具体目标。
- **目标类型**：职业、创业、学习、财富、健康等。
- **期望时间范围**：达到目标的预期时间或阶段。
- **真实动机**：目标背后的内在动机和期望收益。

### 3. 已知背景

总结用户个人背景、资源和约束，可按照“背景挖掘参考”中的类别整理。

### 4. 目标合理性判断

分析目标是否清晰、真实、值得投入、适合当下阶段，指出最大的不确定性。

### 5. 外部现实分析

根据现实校验的结果，说明外部环境中的机会、阻力、时间窗口、可复制性与壁垒、合规与伦理风险、必备资源与外部依赖等。指出需要进一步验证的外部信息。

### 6. 可选路径

详细描述不同路径的适用条件、主要动作、预期收益和风险。通常包括：

#### 路径 A：稳健路径
- …

#### 路径 B：进取路径
- …

#### 路径 C：低成本试错路径
- …

也可以添加其他路径，如外部合作路径、自我提升路径等，视情况而定。

### 7. 推荐路径与理由

明确推荐采用哪条路径（或组合路径），说明选择的理由、成立的前提和最大风险。对于不建议采用的路径，应解释原因。

### 8. 个人适配分析

总结个人适配阶段的发现：

- 用户适合的执行方式（节奏、监督、反馈）。
- 不适合用户的路径或执行方式及原因。
- 需要建立的外部约束或监督机制。
- 需要养成的习惯与应对策略。

### 9. 批判审查总结

整理批判审查阶段的主要结论：

- 最强反方观点。
- 可能高估的地方。
- 可能低估的地方。
- 可能的自欺行为。
- 最危险的隐藏假设。
- 失败高概率原因。
- 必须设置的止损条件。

### 10. 执行计划

根据执行计划阶段，说明未来 2 周、1 个月、3 个月的行动步骤和关键指标，复盘节奏、风险预案等。

### 11. 指标与复盘机制

- **输入指标**：投入的资源和行动次数。
- **过程指标**：任务完成率、进度、用户参与等。
- **结果指标**：收益、转化率、满意度、成本效益等。
- **复盘节奏**：每周或每阶段复盘的频率和重点。

### 12. 下一步建议

给出明确的下一步行动和建议，例如：

- 立即进行几次用户访谈或调研。
- 制作原型或示范，收集反馈。
- 尝试低成本试错路径验证核心假设。
- 调整目标或路径，或推迟执行直到条件成熟。

通过使用本模板，可以将之前的分析成果整理成系统化的报告，既便于自身复盘，也方便与团队或利益相关方沟通决策。
